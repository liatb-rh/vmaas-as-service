# Specs and acceptance criteria

Use **`TEMPLATE.md`** as the starting point for each feature or epic. Prefer **stable AC IDs** (`AC-1`, `AC-2`, …); add new IDs rather than renumbering.

## Rules

- Each **AC** must be **verifiable** (API response, UI behavior, metric, log, or e2e flow).
- Map PRs and tests back to **AC-x** in descriptions or comments.
- **QA Agent** maps tests to these ACs; **Review Agent** checks PRs against them.
